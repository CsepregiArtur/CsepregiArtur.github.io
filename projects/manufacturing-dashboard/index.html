Capabilities
Key Features
📊

Real-Time KPI Dashboard

Total orders, active/planned/unplanned, production hours, late orders, completion rates — all with server-side filtering.

📅

Gantt Timeline View

Per-workstation production timeline with calculated start/end times based on TOP/BOTTOM assembly minute standards and intelligent fallback.

🔄

Snapshot Versioning

Every CSV import creates a timestamped snapshot. Full planning history preserved for audit, comparison, and trend analysis.

🔍

Change Tracking

Automatic detection of 9 change types between consecutive snapshots. Know exactly what changed, when, and how.

🔥

Workstation Heatmap

Workstation × Day matrix showing production intensity and hours. Identify bottlenecks at a glance.

📥

Automated Import Scheduler

Background thread with configurable interval. Auto-detects new CSV files and imports them without user intervention.

⏱️

Production Time Estimation

CTE-based SQL queries using TOP/BOTTOM assembly minute standards with intelligent fallback to 0.5 min/piece.

💾

Auto Database Backup

Automatic integrity-checked backup every 2 hours. SQLite WAL mode for concurrent read safety.

Technology
Tech Stack
Python 3.x Flask 3.0.0 Waitress 2.1.2 SQLite (WAL) JavaScript ES6+ ApexCharts.js PyInstaller JWT/RS256
Frontend: Single HTML file — no framework, no npm, no build step. Vanilla JS + ApexCharts with local bundling for offline operation.

Backend: Flask REST API served by Waitress WSGI server. 20 JSON endpoints with global server-side filtering. Background import scheduler + auto-backup.

Distribution: Self-contained PyInstaller executable with embedded Python, Flask, SQLite, and frontend. Single-file deployment with JWT/RS256 license validation and hardware binding.

Data Model
Database Schema
snapshots — id, moment_salvare, nota
production_plan — id, snapshot_id(FK), workstation, order_number, material_code, description, scheduled_timestamp, is_active, is_planned, sequence, pp_platform, qty_launched, base_finish_date, sap_planned_finish, ... (14+ columns)
platforms_master — id, pp_key, platform_name, min_top, min_bottom, count_top, count_bottom
change_tracking — id, from_snapshot_id(FK), to_snapshot_id(FK), change_type, workstation, order_number, old_value, new_value, old_sequence, new_sequence, (9 change type tracking fields)
evaluation_results — id, snapshot_id(FK), section, workstation, order_count, platform_name
